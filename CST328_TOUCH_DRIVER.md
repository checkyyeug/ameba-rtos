# CST328 触摸驱动说明

## 硬件信息

| 信号 | 引脚 | 说明 |
|------|------|------|
| SDA | PB_10 | I2C2 数据线 |
| SCL | PB_11 | I2C2 时钟线 |
| INT | PA_9 | 中断，低电平有效，有触摸时拉低 |
| RST | PA_10 | 复位，低电平有效 |
| LCD_RESET | PA_14 | LCD 复位（显示驱动使用，与触摸无关） |

- **I2C 地址**：`0x1A`（7-bit，驱动会自动探测 0x1A / 0x15）
- **I2C 速率**：400 kHz（Fast Mode）
- **最大触点数**：5 点

## CST328 寄存器协议（来自官方 Datasheet v2.2）

### NORMAL mode 触摸数据寄存器（起始地址 0xD000）

每个手指占 **7 字节**，最多 5 个手指（共 35 字节）：

| 偏移 | 寄存器 | 内容 |
|------|--------|------|
| +0 | 0xD000 | `ID[7:4]` \| `status[3:0]`（0x06=按下，其他=抬起） |
| +1 | 0xD001 | X 坐标高 8 位（`X_pos >> 4`） |
| +2 | 0xD002 | Y 坐标高 8 位（`Y_pos >> 4`） |
| +3 | 0xD003 | `X 低 4 位[7:4]` \| `Y 低 4 位[3:0]` |
| +4 | 0xD004 | 压力值 |
| +5 | 0xD005 | `key_flag[7]` \| `finger_num[3:0]`（总触点数） |
| +6 | 0xD006 | 固定值 0xAB |

第 2 个手指从 `0xD007` 开始，依此类推。

### 坐标计算公式

```c
X = ((uint16_t)buf[o+1] << 4) | (buf[o+3] >> 4);
Y = ((uint16_t)buf[o+2] << 4) | (buf[o+3] & 0x0F);
```

其中 `o = i * 7`（i 为手指索引，0~4）。

### 触点状态判断

```c
status = buf[o+0] & 0x0F;
is_touching = (status == 0x06);   // 0x06 = 按下
```

### 读取流程

1. 检测 INT 引脚：`GPIO_ReadDataBit(PA_9) == 0` 才读（低电平有效）
2. 从 `0xD000` 起读 35 字节（burst read）
3. 解析 `buf[5]` 得到 `finger_num`
4. 按每 7 字节一组解析各手指坐标

## 坐标映射（物理竖屏 → LVGL 横屏）

### 物理屏坐标系

CST328 报告的是**物理竖屏坐标**：
- 物理分辨率：480（宽）× 800（高）
- raw_x 范围：0（右边）~ 479（左边）**（X 轴方向已翻转）**
- raw_y 范围：0（顶部）~ 799（底部）

### LVGL 逻辑坐标系

LVGL 配置为**横屏**：800（宽）× 480（高），使用 `ROTATION_90`

### 映射公式

```c
LVGL_x = raw_y;                  // 0~799
LVGL_y = (480 - 1) - raw_x;     // 0~479，X 轴翻转补偿
```

### 四角验证数据（实测）

| 物理位置 | raw_x | raw_y | LVGL_x | LVGL_y |
|----------|-------|-------|--------|--------|
| 左上角 | ~441 | ~9 | ~9 | ~38 |
| 右下角 | ~19 | ~762 | ~762 | ~460 |

## Ameba I2C API 注意事项

- `i2c_write(obj, addr, data, len, stop)` — `addr` 直接传 **7-bit 地址**，无需左移
- `stop=0`：发 Repeated Start（用于写寄存器地址后接读）
- `stop=1`：发 STOP（读数据结束）
- 返回值为实际传输字节数，读/写成功时应等于 `len`
